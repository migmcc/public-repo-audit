# v0.2 Plan: Output Format Options

Issue: [#3 Add JSON-only and Markdown-only output options](https://github.com/migmcc/public-repo-audit/issues/3)

## Decision

Start v0.2 with this issue because it is small, backward-compatible, and does not alter scoring, findings, report schemas, repository profiles, or release-readiness rules.

## Status

Implemented on branch `feat/output-format-options`.

## Objective

Allow users to choose which report format is written while preserving the v0.1 default behavior.

## CLI Interface

Add one option to the existing command:

```text
public-repo-audit <path> --format markdown
public-repo-audit <path> --format json
public-repo-audit <path> --format both
```

Recommended argparse shape:

```text
--format {markdown,json,both}
```

## Default Behavior

Default remains:

```text
--format both
```

Therefore this command must keep writing both default files:

```text
public-repo-audit .
```

Expected default outputs remain:

- `report.md`
- `report.json`

## Compatibility Requirements

- Do not remove or rename `--markdown`.
- Do not remove or rename `--json`.
- Do not change the Markdown report content.
- Do not change the JSON report schema.
- Do not change scoring, verdicts, blockers, warnings, recommendations, or checklist logic.
- Existing scripts that call `public-repo-audit <path>` must keep working.

## Expected Behavior

| Command | Markdown output | JSON output |
|---|---|---|
| `public-repo-audit .` | yes | yes |
| `public-repo-audit . --format both` | yes | yes |
| `public-repo-audit . --format markdown` | yes | no |
| `public-repo-audit . --format json` | no | yes |
| `public-repo-audit . --format markdown --markdown custom.md` | `custom.md` | no |
| `public-repo-audit . --format json --json custom.json` | no | `custom.json` |

## Test Plan

Add focused tests for CLI/report routing only.

Recommended tests:

1. Default still writes both `report.md` and `report.json`.
2. `--format both` writes both files.
3. `--format markdown` writes only the Markdown file.
4. `--format json` writes only the JSON file.
5. Custom `--markdown` path is respected in Markdown-only mode.
6. Custom `--json` path is respected in JSON-only mode.
7. Invalid format values fail through argparse with a non-zero exit.

Existing audit/scoring tests should remain unchanged.

## Implementation Notes

Likely files to touch during implementation:

- `src/public_repo_audit/cli.py`
- `tests/test_audit.py` or a new focused `tests/test_cli.py`
- `README.md`
- `CHANGELOG.md`

Suggested implementation approach:

1. Add `--format` to `build_parser()` with `choices=("markdown", "json", "both")` and default `"both"`.
2. In `run()`, call `write_markdown_report()` only when format is `markdown` or `both`.
3. In `run()`, call `write_json_report()` only when format is `json` or `both`.
4. Keep summary printing unchanged.

## Documentation Updates

README should document:

```powershell
public-repo-audit . --format markdown
public-repo-audit . --format json

```

CHANGELOG should add an unreleased v0.2 note only when implementation begins.

## Out Of Scope

- New output formats.
- Changing output filenames by default.
- Changing report schemas.
- Changing report contents.
- Changing scoring behavior.
- Adding config files.
- Creating a new release or tag.

## Risk

Low. Main risk is accidentally breaking default output compatibility. The test suite should explicitly protect the default two-report behavior.

## Definition Of Done For Implementation Run

- CLI supports `--format markdown`, `--format json`, and `--format both`.
- Default still writes both reports.
- Tests cover all output modes.
- README documents the new option.
- `ruff check .` passes.
- `pytest` passes.
- `public-repo-audit .` still returns `100/100`, `showcase-ready`, `0` blockers for this repository.
